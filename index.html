Abstract

Large language models (LLMs) and autonomous coding agents are increasingly involved in software maintenance tasks, including code refactoring. Among these tasks, code clone removal is suited for automation. However, little is known about how AI agents actually perform clone refactoring in real-world development workflows compared to human developers. This paper presents an empiri- cal study of 135 clone-related pull requests (PRs) from the AIDev dataset, equally divided into three groups: (1) PRs created and writ- ten by humans, (2) PRs created by humans but generated by agents, and (3) PRs created and written entirely by agents. Our quantita- tive analysis shows that agent-involved PRs are completed up to 20√ó faster (median closure time 1.0 vs. 15.3 hours; ùõø = 0.40‚Äì0.59, p < 0.001), with 50‚Äì70% smaller change sizes (median churn 90 vs. 202 lines) and 30‚Äì55% lower reviewer workload. For refactoring strategies, we found 75 cases of direct removal, 51 cases of extrac- tion refactoring, and 8 cases of pull-up refactoring (only in agent PRs). Overall, our findings show that AI agents are effective for clone refactoring tasks requiring precision and speed, but they still rely on human oversight when refactoring involves multiple files and requires understanding the broader project structure.

Python Scripts

The scripts are inside the Scripts/ folder. Requirements.txt includes the necessary Python packages. It is in the Scripts folder as well. Install: pip install -r requirements.txt

  • get_dataset.py ‚Äî Download a table from the AIDev dataset and save it as CSV and a SQLite DB.
    Run: python get_dataset.py (edit the table variable inside the file to choose the table).
  • get_clone_data.py ‚Äî Find clone‚Äërelated pull requests in a CSV using keyword matching and write matches to review_clone.csv in the output folder.
    Run: python get_clone_data.py --file all_pull_request.csv --out_dir out/
  • get_code_human.py ‚Äî Download code changes for human‚Äëwritten PRs from GitHub (commits, files, diffs).
    Run: python get_code_human.py
  • get_refactor.py ‚Äî Find refactor‚Äëtagged PRs from pull_request.csv and save subsets as CSV/DB files under refactor_data/curated_agent/.
    Run: python get_refactor.py
  • pr_code_scraper.py ‚Äî Helper to fetch code artifacts for PRs via the GitHub API (used by other scripts or standalone).
    Run: python pr_code_scraper.py or import it in Python.
  • merge_sqlite_dbs.py ‚Äî Merge many SQLite DBs in a folder into a single database.
    Run: set source_folder and output_database at the top of the file, then run python merge_sqlite_dbs.py.
  • run_rminer.py ‚Äî Generate or execute RefactoringMiner commands for selected PR IDs.
    Run (print commands): python run_rminer.py --csv human_pull_request.csv --ids 2625652637,2474497456 --out /tmp/rminer.tsv
    Run (execute): python run_rminer.py --csv human_pull_request.csv --ids-file pr_ids.txt --rminer /path/to/RefactoringMiner --results-dir rminer_results
  • get_pr_metrics.py ‚Äî Calculate quantitative metrics for pull requests, including size, time to close, comments, and churn, from raw PR data. Outputs a summary CSV file.
    Run: python get_pr_metrics.py --input pull_request.csv --output metrics.csv
  • compare_pr_metrics.py ‚Äî Compare PR metrics between human, hybrid, and agent groups using Mann‚ÄìWhitney U tests and effect size calculations. Generates result tables for statistical analysis.
    Run: python compare_pr_metrics.py --input metrics.csv --output results.csv

Experimental Data

The experimental data used in this study are derived from the AIDev dataset and its related GitHub mining results. The data can be downloaded here. Below are brief descriptions of each dataset:

  • all_pull_requests ‚Äî A large master dataset containing metadata for all pull requests across repositories, including IDs, descriptions, timestamps, and repository links.
  • all_repositories ‚Äî Contains metadata about all repositories in the dataset, including repository ID, name, owner, and project language.
  • all_user ‚Äî Includes detailed user information such as user ID, username, and account type (human or agent).
  • human_pull_request ‚Äî The main dataset of pull requests created by humans. Each record includes PR ID, title, description, timestamps, and repository linkage.
  • human_pr_task_type ‚Äî Provides task classification for each human PR, such as bug fix, refactor, or feature addition.
  • issue ‚Äî Lists issues linked to PRs, including issue titles, descriptions, and repository references.
  • pr_comments ‚Äî Contains textual comments left by contributors and reviewers within PR discussions.
  • pr_commits ‚Äî Includes commit-level details for each PR, such as commit IDs, timestamps, and commit messages.
  • pr_review_comments ‚Äî Contains review comments made for curated PRs during code review.
  • pr_reviews ‚Äî Contains general review metadata for each pull request, including reviewer IDs, states, and timestamps.
  • pr_task_type ‚Äî Lists task categories of pull requests, such as feature, bug fix, or refactor.
  • pr_timeline.csv ‚Äî Records key timeline events for each pull request, including creation, first review, and merge times.
  • curated_pull_request ‚Äî Includes all pull request metadata across the dataset, such as ID, title, author, and status.
  • related_issue ‚Äî Maps curated pull requests to their related issues, linking their respective IDs.
  • repository ‚Äî Contains detailed curated repository-level information including names, languages, and creation times.
  • user ‚Äî Provides user information for curated PRs such as user IDs, usernames, and account attributes.
  • mined_code - Includes the code artifacts (commits, files, diffs) for related human PRs and curated PRs.
  • clone_related_data - Includes the clone-related PRs for title, body and both identified from the AIDev dataset using keyword matching.
